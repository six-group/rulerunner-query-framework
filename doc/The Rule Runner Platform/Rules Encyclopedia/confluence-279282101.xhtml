This page lists the rules written for the Rule Runner platform. Private rules may be described here as well.

Important!

When adding new rules in the tables below, please sort them by name!

Naming convention

Rule Runner rules are named in all lowercase, using the underscore as word separator. A prefix of either six_ or userid_ is used depending on the rule type:

  • Official rules: six_
    Example: six_generic_query
  • Inofficial rules: userid_
    Example: txyz_debug_ssl

Allows to directly run Beanshell code in the application server and display its print output and/or the returned object. Can be used to experiment with the IIQ API or JDK methods.
