The Generic Query, available directly in the SIX IAM Web GUI, is the Swiss Army knife of the IIQ operator concerning investigations of various kinds. It is a flexible data query and analysis tool based on the IIQ object model (Reference) and the Object Query Framework, particularly OrionQL (Reference) and CerberusLogic (Reference), and offers powerful query and data transformation capabilities that are not available in plain SQL.

In addition to being used as a query tool, the interactive playground the Generic Query offers makes it a great learning tool - not only for the two DSLs but also for the IIQ object model itself and the rather hidden IIQ internals. It can also be used to debug DSL code used in other SIX IAM components like the Automatic Reconciliation.

While simple queries can easily be done without any knowledge of the DSLs and with only basic knowledge of the IIQ object model, to unleash the full power of the Generic Query, a good understanding of the two DSLs as well as the IIQ object model will be beneficial. However you can start from zero and grow incrementally, using the Generic Query as a learning tool and referring to the three reference documents linked above. After having done your first steps, it will be probably advantageous to read the two introductory articles for the DSLs, A Gentle Introduction to OrionQL Expressions and A Gentle Introduction to CerberusLogic to get familiar with the possibilities they offer. Mastering the Generic Query will let you work wonders!

To access the Generic Query, on any SIX IAM instance go to /identityiq/rulerunner/rulerunner.jsf?rule=six_generic_query. As there is currently no authorization concept, the access will require administrator rights. From this page you can create a browser bookmark by submitting the empty form (ignoring the resulting error message) and then dragging the link that appeared to the right of the Execute button to your browser bookmarks. (This bookmark is actually a Bookmarklet and will work for all SIX IAM instances by taking the host from the current page and opening the contained path in a new browser tab.)
